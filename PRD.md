# AutoLogin 校园网自动登录 — 产品需求文档（PRD）

## 1. 项目信息

| 字段 | 内容 |
|------|------|
| **项目名称** | `autologin` |
| **语言** | 中文 |
| **技术栈** | C++17 + Qt6 QML |
| **架构模式** | MVVM |
| **原始需求** | 将 Windows 桌面校园网自动登录守护程序从 C + Qt Widgets 重构为 C++17 + Qt6 QML 架构，保留全部现有功能，提升代码质量与 UI 体验 |

---

## 2. 产品定义

### 2.1 产品目标

1. **零感知联网**：校园网认证全自动化，用户开机后无需任何操作即可联网，包含定时重认证与断线恢复
2. **现代桌面体验**：通过 QML 声明式 UI + 毛玻璃视觉风格 + 流畅动画，提供远超原版 Widgets 的交互质感
3. **可维护可扩展**：消除全局变量与 C 风格代码，采用 MVVM 架构与现代 C++17 特性，使后续功能迭代（如多校园支持、云端配置同步）成本最小化

### 2.2 用户故事

| # | 用户故事 |
|---|---------|
| US1 | 作为住校学生，我希望开机后自动完成校园网认证，这样我不必每天手动打开登录页面 |
| US2 | 作为住校学生，我希望断网后程序自动重连，这样我在游戏/视频时不会因网络掉线而中断 |
| US3 | 作为住校学生，我希望认证成功后自动开启移动热点，这样我的手机/平板可以无缝共享网络 |
| US4 | 作为住校学生，我希望通过托盘图标快速了解当前联网状态，这样我不用打开浏览器测试 |
| US5 | 作为住校学生，我希望在设置窗口中调整认证间隔和偏移，这样我可以避开校园网维护时段 |

## 3. 技术规范

### 3.1 需求池

#### P0 — 必须实现（Must Have）

| ID | 需求 | 说明 |
|----|------|------|
| P0-01 | 校园网 HTTP 认证 | POST 请求到网关（默认 172.30.255.2），携带用户名/密码/MAC/IP，完成 Portal 认证 |
| P0-02 | 互联网连通性检测 | HTTP GET `connect.rom.miui.com/generate_204`，仅 204 响应视为连通 |
| P0-03 | 飞行模式检测 | 通过 `GetAdaptersAddresses` 检测活跃非回环网卡，无活跃网卡则跳过认证 |
| P0-04 | 定时重认证 | 每 N 小时（1–24）± 偏移分钟（±30）自动重新认证 |
| P0-05 | 认证防抖 | 原子锁 + 3 秒时间窗口，防止并发重复认证请求 |
| P0-06 | 黑名单时段 | 03:55–05:05 禁止发起认证（校园网维护时段） |
| P0-07 | 指数退避重试 | 失败后等待 `min(300, 5 × 2^fails)` 秒再重试 |
| P0-08 | 系统托盘图标 | 托盘图标 + tooltip 显示当前联网状态 |
| P0-09 | 右键菜单 | 显示日志 / 立即登录 / 清理日志 / 开机自启(可勾选) / 设置 / 退出 |
| P0-10 | 设置持久化 | 配置项持久化存储（注册表 `HKCU\Software\AutoLogin\Settings`） |
| P0-11 | 开机自启 | 注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` 写入/移除 |
| P0-12 | 日志管理 | 自动清理 7 天以上旧日志 + 手动清理按钮 |
| P0-13 | 设置窗口 | 卡片式选项布局，包含所有核心配置项 |
| P0-14 | 二次校验 | 认证响应未匹配关键词时，等 1 秒后再次检测连通性 |

#### P1 — 应该实现（Should Have）

| ID | 需求 | 说明 |
|----|------|------|
| P1-01 | 移动热点自动开启 | 通过 WinRT COM（combase.dll）调用 `NetworkOperatorTetheringManager`，认证成功后自动开热点 |
| P1-02 | WLAN 适配器检测 | 检测无线网卡是否存在，影响热点功能可用性判断 |
| P1-03 | 系统通知 | Shell_NotifyIcon 气泡通知，区分成功/失败/信息三种类型 |
| P1-04 | 左键点击托盘触发认证 | 可配置开关，单击托盘图标立即触发认证 |
| P1-05 | 明暗主题切换 | 太阳/月亮图标切换，带动画过渡效果 |
| P1-06 | 毛玻璃风格 UI | 设置窗口采用 Acrylic/Mica 风格半透明背景 |

#### P2 — 可以实现（Nice to Have）

| ID | 需求 | 说明 |
|----|------|------|
| P2-01 | 提高线程优先级 | 可配置选项，提升守护线程调度优先级以减少认证延迟 |
| P2-02 | 认证网关自定义 | 支持用户修改认证网关地址，为多校园适配预留接口 |
| P2-03 | 日志查看窗口 | 独立的 QML 日志查看器，支持搜索和按级别过滤 |
| P2-04 | 密码加密存储 | 使用 Windows DPAPI 加密存储密码，替代明文注册表存储 |

## 5. 架构约束（对架构师的关键输入）

| 约束 | 说明 |
|------|------|
| C++17 | 必须使用 `std::optional`、`std::variant`、结构化绑定、`if constexpr` 等特性 |
| Qt6 QML | UI 层全部使用 QML 声明式语法，禁止混用 Qt Widgets |
| MVVM | Model（业务数据）→ ViewModel（Q_PROPERTY 暴露给 QML）→ View（QML） |
| 消除全局变量 | 所有原 C 全局状态（`g_auth_busy`、配置等）封装为类成员 |
| Win32 API 隔离 | `GetAdaptersAddresses`、注册表操作、Shell_NotifyIcon 等封装到独立 platform 层 |
| WinRT COM 隔离 | 热点管理封装到独立模块，延迟加载 `combase.dll` |

## 6. 待确认问题

| # | 问题 | 影响 | 建议默认值 |
|---|------|------|-----------|
| Q1 | 是否需要支持多校园网关（不同学校不同网关地址）？ | 影响 P2-02 的优先级和配置结构设计 | 默认硬编码单一网关 |
| Q2 | 密码是否需要加密存储？当前明文存注册表有安全风险 | 影响 P2-04 是否升为 P1 | 建议 P2，后续版本迭代 |
| Q3 | 是否需要日志查看窗口？原版通过外部程序查看日志文件 | 影响 P2-03 的开发工作量 | 默认 P2，首版用系统文件管理器打开日志目录 |
| Q4 | 黑名单时段（03:55-05:05）是否需要可配置？ | 影响设置 UI 是否增加时段配置项 | 默认硬编码，可后续开放配置 |
| Q5 | QML 托盘图标在 Qt6 中需要 `QSystemTrayIcon` + 旧式菜单，还是接受第三方 QML 托盘组件？ | 影响托盘功能实现方式 | 使用 Qt6 原生 `QSystemTrayIcon` + QML 菜单桥接 |
| Q6 | 毛玻璃效果是否需要降级方案（Win10 早期版本不支持 Mica）？ | 影响 UI 兼容性策略 | Acrylic 作为备选，最差降级为半透明纯色背景 |
